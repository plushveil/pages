    Getting Started

    On this page you'll find instructions on how to install the Pages package and how to set up GitHub actions to deploy your website.

    How to Install the Pages Package

    Follow the instructions below to install the Pages package from GitHub:

    1. Install Node.js:Ensure you have Node.js installed on your machine. You can download it from the official Node.js website.
    2. Open Your Terminal:Launch your terminal or command prompt.
    3. Run the Installation Command:
      npm install @plushveil/pages
    4. Verify Installation:After installation, verify that the package is added to yournode_modulesdirectory and listed in yourpackage.jsonfile.
    5. Add thepagescommand to Yourpackage.jsonFile:Add the following line to yourscriptssection in yourpackage.jsonfile:
      {
  "scripts": {
    "pages": "pages"
  }
}

    You can now usenpm run pages -- serve .to serve your website locally.

    Using the Pages GitHub Action

    This setup allows you to automate the deployment process, making it easier to keep your site updated with every change you make.

    To configure the Pages GitHub Action, you'll need to include it in your workflow YAML file (usually found in the.github/workflows/directory of your repository). For more details, you can refer to theGitHub Actions documentationand thePages GitHub Action repository.

    Below is an example configuration for the Pages GitHub Action:

    Example Configuration

    name: Build and Deploy

on:
  push:
    branches:
      - main

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout code
        uses: actions/checkout@v2

      - id: build
        uses: plushveil/pages@latest
        with:
          folder: '.'
          config: 'pages.config.mjs'
        env:
          HOST: website.example
          PORT: 443
          PATHNAME: /

      - id: deploy
        run: |
          cp -r ${{steps.build.outputs.folder}}/* .

    Breakdown of the GHA steps

    1. Checkout Step:Usesactions/checkoutto retrieve the repository code.
    2. Build Step:Specifies the Pages action withuses: plushveil/pages@latest, which will build your website. Thewithblock defines the folder where your static site files are located. Theenvblock SetsHOST,PORT, andPATHNAME.
    3. Deploy Step:Copies the built files from the Pages actions output folder to the root directory.

    Once you have set up the above configuration in your workflow file, the GitHub Action will automatically run whenever you push changes to themainbranch. This action will not deploy your site to a server, but it will build the static files that you can then deploy to your server yourself.
